#  Race Simulator

Консольный симулятор гонки на Python. Пользователь задаёт количество пилотов, их
характеристики и длину гонки, а программа круг за кругом рассчитывает время
прохождения с учётом износа шин, расхода топлива, DRS и пит-стопов, и в конце
выводит итоговую турнирную таблицу.

## Возможности

- **Расчёт времени круга** с учётом мощности мотора, скорости, опыта пилота,
  износа шин и веса топлива.
- **Износ шин** (Soft / Medium / Hard), который ускоряется, если гонщик едет
  плотно за соперником («грязный воздух»).
- **DRS** — бонус к скорости, если разрыв до впереди идущего гонщика меньше 1 секунды.
- **Автоматические пит-стопы**, когда шины изнашиваются ниже 30% (с выбором
  нового типа шин в зависимости от оставшихся кругов).
- **Расход и контроль топлива** — гонщик сходит с дистанции (DNF), если топливо заканчивается.
- Итоговый протокол: финишировавшие пилоты по местам + список сошедших.
- Поддержка json файлов: можно загружать свои списки гонщиков

## Требования

- Python 3.7+
- Никаких внешних зависимостей (используется только `random`, `json`, `os` из стандартной библиотеки)

## Запуск

```bash
python race.py
```

Программа запросит данные в интерактивном режиме.

## Входные данные

**Общие параметры гонки:**

| Параметр | Описание |
|---|---|
| Количество гонщиков на старте | Сколько пилотов участвует |
| Количество кругов | Длина гонки |

**Для каждого гонщика:**

| Параметр | Описание |
|---|---|
| Имя пилота | Произвольная строка |
| Мощность мотора | Число, ориентир — 850 |
| Стартовые шины | `Soft`, `Medium` или `Hard` |
| Рейтинг скорости | 0–100 |
| Рейтинг опыта | 0–100 |
| Расход топлива за круг | Дробное число, например 3.2 |
| Качество пит-стопов команды | 0–100 (влияет на время пит-стопа) |

Начальный запас топлива генерируется автоматически (случайно, в зависимости от числа кругов).

## Как устроена механика

### Время круга (`calculate_laptime`)
Базовое время круга (70 сек) корректируется:
- **мощностью мотора** — чем мощнее, тем быстрее;
- **износом шин** — чем сильнее износ, тем больше потеря темпа (разный коэффициент для Soft/Medium/Hard);
- **скоростью и опытом** пилота — снижают время круга;
- **весом топлива** — чем больше топлива в баке, тем медленнее едет болид.

После расчёта времени круга топливо уменьшается на величину `fuel_management`.

### Износ шин (`tire_degradation`)
Скорость износа зависит от типа шин и количества кругов на них (чем старше
резина, тем быстрее она "умирает"), а также от разрыва (`gap`) до соперника
впереди — плотная езда в "грязном воздухе" увеличивает износ.

### DRS
Активируется, если разрыв до впереди идущего активного гонщика не превышает
1 секунды — даёт скидку 0.6 сек к времени круга.

### Пит-стопы (`make_pitstop`)
Срабатывают автоматически, когда износ шин опускается ниже 30% (и до финиша
остаётся больше 6 кругов). Тип новой резины выбирается по правилам:
- Soft → Hard (если кругов много) или Medium
- Medium → Hard (если кругов много) или Soft
- Hard → Medium (если кругов много) или Soft

Пит-стоп добавляет к общему времени гонщика ~20–23 секунды (зависит от
качества пит-стопов команды).

### Сход с дистанции
Если запас топлива становится меньше расхода за круг, гонщик считается сошедшим (DNF).

```
[12] Verstappen заехал на пит-стоп и переобулся в Medium!
[34] === Sainz СХОД: Закончилось топливо! ===

1. Verstappen | Итоговое время: 84м 12.345с
2. Norris | Итоговое время: 84м 20.101с

--- НЕ ФИНИШИРОВАЛИ (DNF) ---

```

## Возможные доработки

- Вынести константы (базовое время круга, коэффициенты износа и т.д.) в конфиг.
- Добавить случайность/погодные условия.
- Сохранение результатов в файл (CSV/JSON).
- Валидация пользовательского ввода (сейчас `Soft/Medium/Hard` и границы 0–100 не проверяются).
