# Race Simulator

Консольный симулятор гонки на Python. Пользователь задаёт длину гонки и
характеристики пилотов (вручную или загружает их из JSON-файла), а программа
круг за кругом рассчитывает время прохождения с учётом износа шин, расхода
топлива, обгонов, ошибок пилотов, поломок, машины безопасности и пит-стопов.
В конце выводится итоговый протокол гонки.

## Возможности

- Расчёт времени круга с учётом мощности мотора, скорости, опыта пилота,
  износа шин, веса топлива и случайного разброса (тем меньше у пилота опыта,
  тем нестабильнее его темп).
- Износ шин (Soft / Medium / Hard), ускоряющийся при плотной езде за соперником
  ("грязный воздух"), а также замедляющийся в режиме безопасности.
- DRS-бонус при разрыве до 1 секунды и система обгонов: удачный обгон,
  ошибка обороняющегося, контакт с потерей времени, двойной сход после аварии,
  или просто "застревание" за соперником.
- Случайные сходы с трассы (поломки, ошибки пилота) с вероятностью, растущей
  по ходу гонки.
- Режимы безопасности: Virtual Safety Car (VSC) при одном сходе, Safety Car
  (SC) при столкновении двух машин, красный флаг (RF) при массовом завале —
  гонка останавливается.
- Автоматические пит-стопы при падении износа шин ниже 30%, с шансом
  заминки на пит-стопе (зависит от качества работы команды).
- Контроль топлива — гонщик сходит с дистанции, если топливо заканчивается.
- Загрузка состава пилотов из `drivers.json` либо ручной ввод, если файла
  нет или пользователь отказался его использовать.
- Два режима вывода: подробный отчёт по каждому кругу (`First`) или сразу
  финальные результаты без промежуточных событий (`Second`).
- Статистика по итогам гонки: финишный протокол, список сошедших, лучший
  круг гонки, лучший пит-стоп.

## Требования

- Python 3.7+
- Только стандартная библиотека (`json`, `os`, `time`, `random`)

## Запуск

```bash
python race.py
```

Программа запросит данные в интерактивном режиме.

## Входные данные

**Общие параметры:**

| Параметр | Описание |
|---|---|
| Количество кругов | Длина гонки |
| Режим отображения (First/Second) | Подробный вывод по кругам или только итог |

**Для каждого гонщика (если вводится вручную):**

| Параметр | Описание |
|---|---|
| Имя пилота | Произвольная строка |
| Название команды | Произвольная строка |
| Мощность мотора | Число, ориентир — 850 |
| Стартовые шины | `Soft`, `Medium` или `Hard` |
| Рейтинг скорости | 0-100 |
| Рейтинг опыта | 0-100 |
| Расход топлива за круг | Дробное число, например 3.2 |
| Качество пит-стопов команды | 0-100 (влияет на риск заминки и её время) |

Начальный запас топлива генерируется автоматически (случайно, в зависимости
от числа кругов).

### Загрузка из JSON

Если в рабочей директории есть файл `drivers.json`, программа предложит
загрузить состав из него вместо ручного ввода. Ожидаемый формат — список
объектов с полями `name`, `team`, `power`, `tire_type`, `speed`,
`experience`, `fuel_management`, `pit_stop_quality`. При ошибке чтения файла
программа переходит к ручному вводу.

## Как устроена механика

### Время круга (`calculate_laptime`)
Базовое время круга (70 сек) корректируется мощностью мотора, износом шин
(разные коэффициенты для Soft/Medium/Hard), скоростью и опытом пилота, а
также весом топлива в баке. После расчёта к результату добавляется случайный
разброс — чем ниже опыт пилота, тем шире диапазон отклонения. После расчёта
круга топливо уменьшается на величину `fuel_management`.

### Износ шин (`tire_degradation`)
Скорость износа зависит от типа резины и её возраста в кругах, а также от
разрыва (`gap`) до соперника впереди: плотная езда в "грязном воздухе"
увеличивает износ. Во время VSC или SC износ замедляется.

### Сходы с трассы (`retire`)
На каждом круге для активного пилота проверяется случайный сход (поломка
двигателя, гидравлики, коробки передач, прокол или ошибка пилота), вероятность
которого растёт по ходу гонки.

### Обгоны и DRS
Если разрыв до идущего впереди пилота не превышает 1 секунды и нет режима
безопасности, активируется DRS и может произойти одно из событий: успешный
обгон, ошибка обороняющегося пилота, контакт с обоюдной потерей времени,
серьёзная авария с двойным сходом, либо атакующий просто "застревает" за
соперником и теряет темп.

### Машина безопасности (`check_safety_car`)
При одном сходе за круг вводится Virtual Safety Car (снижение скорости на
40%), при двух одновременных сходах — полноценный Safety Car (пелотон
собирается вместе), при трёх и более — гонка останавливается красными
флагами.

### Пит-стопы (`make_pitstop`)
Срабатывают автоматически, когда износ шин опускается ниже 30% (и до финиша
остаётся больше 6 кругов). Тип новой резины выбирается по правилам:

- Soft -> Hard (если кругов много) или Medium
- Medium -> Hard (если кругов много) или Soft
- Hard -> Medium (если кругов много) или Soft

Помимо базового времени пит-стопа, есть шанс заминки — тем выше, чем ниже
качество пит-стопов команды. Время возврата на трассу также зависит от
наличия VSC/SC.

### Сход из-за топлива
Если запас топлива становится меньше расхода за круг, гонщик считается
сошедшим.

## Возможные доработки

- Вынести константы (базовое время круга, коэффициенты износа, вероятности
  событий) в конфиг или отдельный модуль.
- Добавить валидацию пользовательского ввода (типы шин, диапазоны 0-100 и
  т.д. сейчас не проверяются).
- Привести форматирование итогового времени к единому виду (в коде
  используются то `.`, то `:` в качестве разделителя минут и секунд).
- Сохранение результатов гонки в файл (CSV/JSON) по аналогии с загрузкой
  состава пилотов.
